# Igris — The King’s AI System

Igris is a React + Vite frontend with an Express server that provides a single API surface for authentication, chat sessions, multimodal file analysis (Gemini), and code preview.

## Project status
- Frontend: React 19 + Tailwind
- Server: Express + Gemini (via `@google/genai`)
- PWA: manifest + service worker generated by `scripts/pwa-setup.js`
- Code preview: sandboxed iframe for fenced code blocks

---

## Prerequisites
- Node.js (LTS recommended)
- Gemini API key (server-side)

---

## Setup
1. Install dependencies
   ```bash
   npm install
   ```

2. Create configuration
   - Add your Gemini key to the environment:
     - `.env.local` (recommended)
     - or `.env`

   Required variable:
   - `GEMINI_API_KEY`

   Example `.env.local`:
   ```env
   GEMINI_API_KEY=your_key_here
   ```

3. (Optional) Codesandbox token
   - If you use the CodeSandbox runner endpoint, also set:
     - `CODESANDBOX_TOKEN`

---

## Run locally
### Dev mode
```bash
npm run dev
```

This starts the dev server and backend together.

- Frontend served at: `http://0.0.0.0:3000`

---

## Build for production
```bash
npm run build
```

Outputs:
- `dist/` (frontend bundle + bundled server)

---

## Lint / typecheck
```bash
npm run lint
```

---

## PWA (offline / installable)
PWA assets are generated during build/dev startup by:
- `scripts/pwa-setup.js`

It writes:
- `public/manifest.json`
- `public/sw.js`
- icon files into `public/`

If you want to customize the PWA appearance:
- edit `scripts/pwa-setup.js` (manifest values + icon sources)

---

## Notes
### Chat code preview
- Fenced blocks are rendered in `src/components/CodeSnippet.tsx`.
- HTML preview is sandboxed.
- `<script>` tags inside HTML fences are rendered inert (visible, not executed) to keep the preview safe.

---

## Deployment
You can deploy the generated `dist/` output to any Node-capable host.

Typical flow:
1. `npm ci`
2. `npm run build`
3. `npm run start`

(If `start` is not present in your environment, deploy using the server entrypoint produced in `dist/`.)
